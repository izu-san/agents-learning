# E2E 実行ブラウザ/デバイス方針（主要ブラウザ・PC/スマホ）

## 推奨マトリクス（コスト最小で効果最大）
- PRごと（必須）：PC Chromium（Windows）
- 毎晩（推奨）：PC WebKit + PC Firefox（どちらか片方でも可）
- 毎晩（推奨）：スマホ相当 1機種（例：iPhone 14 / Pixel 7）をPlaywrightデバイスエミュレーションで
- リリース前（推奨）：スマホ相当をもう1機種追加（iOS/Androidの両方をカバー）

## なぜこうするか
- まずPRで「最頻出ブラウザ + 主要導線」を高速に落とす
- エンジン差（Chromium/WebKit/Firefox）とモバイルのレイアウト崩れは夜間バッチで拾う

## シナリオの割り当て例
- P0（PR）：A-01/A-02、S-01/S-02、B-02、M-01/M-02
- P1（夜間）：B-01/B-04、N-01/N-02、A-03
- P2（任意）：E-01/P-01

## メモ
- 実機スマホでの検証が必要な要件（カメラ/位置情報/生体認証など）が出たら、エミュレーションと別に「実機スモーク」を追加
