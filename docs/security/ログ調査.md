# AWS Bedrock × Playwright Agent  
## 入力データとログの取り扱いについて

---

## 結論

> **AWS Bedrock では、入力したプロンプトや生成結果はモデルの学習には使われず、  
さらに「ログに残さない構成」も可能。**

---

## 1. Bedrock 側ではどう扱われるか

- AWS Bedrock は  
  **プロンプト・生成結果を保存しない**
- **モデル提供会社（Anthropic）にも共有されない**
- **学習データとしても使用されない**

公式ドキュメント：
https://docs.aws.amazon.com/bedrock/latest/userguide/data-protection.html

---

## 2. では「どこにログが残る可能性があるのか」

残る可能性があるのは **自分のAWS環境だけ**

| 場所 | 内容 |
|------|------|
| CloudTrail | 「誰が Bedrock を呼んだか」（本文は記録されない） |
| API Gateway | 設定次第でリクエスト本文がログに残る |
| Lambda / サーバー | 開発者がログ出力すると本文が残る |

Bedrock や Anthropic 側には **一切残らない**。

---

## 3. ログを残さない構成は可能？

**可能。以下を設定すればプロンプトは保存されない。**

### ① CloudTrail
- Bedrock の入力（プロンプト）はデフォルトで記録されない

### ② API Gateway
- Access Logging を OFF  
または
- Body Logging を OFF

### ③ VPC PrivateLink（推奨）
- Bedrock をインターネット経由で呼ばない
- ネットワークログにも残らない

---

## 4. Playwright Agent との関係

Playwright Agent（Planner / Generator / Healer）は  
Bedrock にプロンプトを送ってコードを受け取るだけ。

この構成では：

> **Playwright に渡した顧客データも  
> Bedrockにもログにも残らない**
